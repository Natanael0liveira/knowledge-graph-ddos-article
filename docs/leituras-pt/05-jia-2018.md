## Resumo

Formas de ciberataque são complexas e variadas, e a detecção e predição de tipos dinâmicos de ataque são sempre tarefas desafiadoras. Pesquisa em grafos de conhecimento está se tornando cada vez mais madura em muitos campos. No presente, é muito significativo que certos acadêmicos tenham combinado o conceito de grafo de conhecimento com cibersegurança para construir uma base de conhecimento de cibersegurança. Este artigo apresenta uma base de conhecimento de cibersegurança e regras de dedução baseadas em um **modelo quintupla**. Usando aprendizado de máquina, extraímos entidades e construímos ontologia para obter uma base de conhecimento de cibersegurança. Novas regras são então deduzidas calculando fórmulas e usando o algoritmo de *path-ranking*. O reconhecedor de entidade nomeada Stanford (NER) também é usado para treinar um extrator para extrair informação útil. Resultados experimentais mostram que o Stanford NER fornece muitas características e o parâmetro *useGazettes* pode ser usado para treinar um reconhecedor no domínio de cibersegurança em preparação para trabalho futuro.

## 1. Introdução

No presente, algumas bases de conhecimento de cibersegurança já foram estabelecidas para certos aspectos do domínio de cibersegurança. Por exemplo, *common vulnerability enumeration* é uma base de dados de vulnerabilidades na qual cada vulnerabilidade recebeu um ID unificado, conforme definido pela MITRE Corporation. Outra informação também está disponível, como nível de ameaça, tipo de ameaça, e assim por diante. Uma base de conhecimento de processo fornece informação básica sobre processos comuns, e alguns conhecidos *vendors* de antivírus estabeleceram enormes bases de assinatura sobre vírus. Adicionalmente, a Internet tornou-se uma fonte primária de conhecimento e informação, e contém uma grande quantidade de conteúdo relacionado a cibersegurança como *blogs* de segurança, fóruns de *hackers* e boletins de segurança. Fazer uso completo de informação relacionada a cibersegurança de uma variedade de bases de conhecimento e *sites*, e juntar todo esse conhecimento relacionado a segurança, será útil para detecção de intrusão e consciência situacional de cibersegurança.

O trabalho principal neste artigo é dividido em duas partes. A primeira parte discute construir uma base de conhecimento de cibersegurança seguindo um procedimento de três passos, e propõe um *framework* para construir a base de conhecimento: Primeiro, obter informação coletando e analisando dados estruturados e não-estruturados; segundo, construir a ontologia de acordo com a informação que foi obtida; e terceiro, gerar a base de conhecimento de cibersegurança. A segunda parte discute dedução de conhecimento de cibersegurança. **Um modelo quintupla de uma base de conhecimento de cibersegurança é proposto para obter novo conhecimento usando o algoritmo de *path-ranking*.**

**O modelo quintupla de base de conhecimento de cibersegurança proposto neste artigo contém os seguintes cinco elementos: conceito, instância, relação, propriedades e regra.** Este modelo fornece uma fundação para construção de ontologia. Aprendizado de máquina é usado para extrair entidades relacionadas a cibersegurança porque o modelo de campo aleatório condicional é adequado para reconhecimento de entidade nomeada. Este artigo usa o Stanford NER para extrair entidades relacionadas a cibersegurança, e usa a implementação base do Stanford NER para treinar um modelo de extração no domínio de cibersegurança. Para verificar a influência de *useGazettes*, construímos três modelos diferentes. Os resultados experimentais mostram que *useGazettes* é importante como meio de treinar um NER no domínio de cibersegurança. A dedução de conhecimento inclui dedução de atributo e dedução de relacionamento. Para dedução de atributo, o novo atributo pode ser obtido usando a fórmula de predição de valor de atributo. Para dedução de relacionamento, o novo relacionamento entre instâncias pode ser obtido usando a fórmula preditiva de raciocínio relacional e o algoritmo de *path-ranking*.

## 2. Trabalhos Relacionados

### 2.1. Construção de Ontologia

Um trabalho significativo foi completado por Undercoffer et al. da Universidade de Maryland, que desenvolveram uma ontologia para modelar ataques e entidades relacionadas. A ontologia proposta é direcionada apenas a ataque. Para representar conceitos e entidades que são relevantes para o domínio de cibersegurança, Joshi et al. propuseram uma ontologia para cibersegurança derivada da ontologia proposta por Undercoffer. Estenderam a ontologia para fornecer relações de modelo que capturam a estrutura de esquema da *US National Vulnerability Database* (NVD) e conceitos de exploração de segurança. Esta ontologia contém 11 tipos de entidade (e.g., vulnerabilidade, produto, meio, consequência). More et al. estenderam a ontologia proposta por Undercoffer e adicionaram regras à lógica de raciocínio.

A MITRE investigou maneiras de desenvolver uma ontologia para o domínio de cibersegurança. Baseado nos esforços de Undercoffer e MITRE, **Iannacone et al. propuseram uma ontologia para uma base de conhecimento de cibersegurança. A ontologia resultante contém 15 tipos de entidade e 115 propriedades no total.**

### 2.2. Extração de Informação

Existem dois métodos principais de extração de conhecimento:

1. **Baseado em engenharia de conhecimento:** depende fortemente de regras de extração, mas pode permitir ao sistema lidar com problemas de extração de informação em domínios específicos. A desvantagem é que tanto profissionais relacionados ao domínio quanto linguistas são requeridos. Pinkston et al. descreveram um sistema no qual uma ontologia foi definida para extrair informação de ataque. Rehman e Mustafa usaram pontuação TFIDF para extrair informação de CVEs.

2. **Baseado em aprendizado de máquina:** o passo básico envolve treinar um modelo de extração de informação usando uma grande quantidade de dados de treinamento. Este método não requer regras definidas por profissionais com antecedência. Lal propôs um sistema que pode identificar entidades relevantes de texto não-estruturado. Mulwad et al. desenvolveram um *framework* para detectar e extrair informação sobre vulnerabilidades e ataques de texto web.

### 2.3. Bases de Conhecimento de Cibersegurança

(1) **Base de dados de vulnerabilidade.** As bases de dados abundantes existentes são CNNVD (China) e NVD (Estados Unidos). Informação inclui nomes, descrições, prioridades, métodos de dano, etc.

(2) **Base de regras de ataque.** Esta base coleta informação sobre ataques existentes. A *Snort attack rule base* é uma base relativamente perfeita.

### 2.4. Raciocínio Baseado em Conhecimento

Raciocínio baseado em conhecimento pode ser dividido em raciocínio baseado em símbolo e raciocínio baseado em estatística. **Raciocínio baseado em símbolo pode inferir uma nova relação entre entidades de um mapa de conhecimento existente usando regras, mas também executar detecção de conflito lógico em um grafo de conhecimento.** Métodos baseados em estatística geralmente usam os métodos de aprendizado de máquina de relacionamento.

A abordagem SDType usa a distribuição estatística de atributos conectados por uma tripla ou predicado para predizer o tipo de uma instância. O método de indução de esquema pode ser subdividido em método baseado em ILP (*inductive logic program*) e método baseado em ARM (*association rules mining*).

### 3.2. Extração de Entidades Relacionadas a Cibersegurança: Um Método Baseado em Aprendizado de Máquina

Um **Conditional Random Field (CRF)** é um modelo de grafo não-direcionado baseado em identificação e segmentação de sequência estatística. Problemas NER podem ser definidos como anotações de sequências.

O modelo CRF é adequado para reconhecimento de entidade nomeada. CRF linear simples é atualmente o melhor método para reconhecimento de entidade nomeada. Modela a distribuição de probabilidade $P(y|x)$:

$$
P(y|x) \propto \prod_{j=1}^{N} \exp\left(\sum_{i=0}^{M} \lambda_i f_i(y_{j-1}, y_j, x_j)\right) \quad (1)
$$

onde $N$ é o número de *tokens* e $M$ é o número de características.

O **Stanford NER** fornece uma implementação geral de modelos de sequência CRF linear (também conhecido como CRFClassifier). **Existem muitas opções para características no Stanford NER.** As características que foram determinadas e usadas para treinar o NER são:

- **UseNGrams:** características de letra n-grams.
- **MaxNGramLeng:** definido como 6.
- **UsePrev/UseNext:** características baseadas em relacionamentos entre palavra atual e <palavra anterior, classe da palavra anterior>.
- **UseWordPairs:** baseado em dois pares de palavras.
- **UseTaggySequences:** sequência de classes em vez de palavras.
- **UseGazettes:** ativa o uso de dicionários de entidades.
- **Gazette:** arquivos com nomes de classes de entidades.
- **CleanGazette:** dispara apenas se a palavra/frase inteira casa.
- **SloppyGazette:** dispara se qualquer *token* da entrada casa.

Este artigo usa a característica gazette para classes software e OS. Construíram um dicionário de entidades a partir da plataforma de campo de influência na base de vulnerabilidade.

## 4. Dedução de Conhecimento

### 4.1. Fonte de Dados

Vulnerabilidade inclui vulnerabilidades existentes que podem ser coletadas. Ataques incluem as técnicas de ataque mais populares hoje. As fontes de vulnerabilidades são CVE, NVD, SecurityFocus, CXSECURITY, Secunia, CNVD, CNNVD e SCAP. As fontes de ataque incluem Pediy BBS, Freebuf, Kafan BBS, OWASP, 360SRC e ASRC.

### 4.2. Análise de Princípio

$K$ é usado para representar o grupo de conhecimento; aqui, $K = \langle$conceito, instância, relação, propriedades, regra$\rangle$, onde:

- **Conceito** = $\{concepto_i, i = 1, \ldots, n\}$. O conceito é um conjunto da ontologia abstrata, como OS, software, ataque, etc.
- **Instância** = $\{instância_i, i = 1, \ldots, m\}$. A instância é um conjunto de exemplos concretos, como Windows 7, Adobe Reader, DDoS, etc.
- **Propriedades** = $\{\langle instância_i, propriedade_{ij}, valor_j \rangle\}$. Um conjunto de valores de atributo de instância.
- **Relação** = $\langle conceito_i, relação_{cc}, conceito_j \rangle | \langle conceito_i, relação_{ci}, instância_j \rangle | \langle instância_i, relação_{ii}, instância_j \rangle$. Mostra o relacionamento entre instâncias, como *subClassOf*, *instanceOf*, *isA*, etc.
- **Regra** = $\{regra | regra = \langle instância_i, nova\_relação_{ij}, instância_j \rangle | ...\}$. As regras são usadas para deduzir novos valores de atributo e novos relacionamentos.

**A parte mais importante do modelo de grupo de conhecimento apresentado neste artigo são as regras.** Essas regras podem ser usadas para deduzir novos relacionamentos e novos valores de atributo.

**Dedução de atributo** usa os atributos existentes da instância e condições para deduzir novos atributos.

**Dedução de relacionamento** usa os relacionamentos entre instâncias existentes para deduzir novos relacionamentos. Por exemplo, dado:

- Relação 1 (Si Li, local de nascimento, Pequim)
- Relação 3 (Pequim, pertence ao país, China)
- Relação 6 (Si Li, colega de classe, San Zhang)
- Relação 7 (San Zhang, local de nascimento, Pequim)

Um novo relacionamento pode ser obtido: **Relação 8 (San Zhang, nacionalidade, Chinês)**.

### 4.3. Resultado da Dedução

#### 4.3.1. Dedução de Atributo

Atributos são representados por pares (nó, chave, valor). A fórmula de predição do valor do atributo:

$$
Valor_{ik} = \sum_{j=1}^{m} \lambda_j \cdot f_{ij}(chave_j, valor_j) + \sum_{t=1}^{l} \sum_{j=1}^{m} r_t \cdot \lambda_j \cdot f_{ij}(chave_j + valor_j) \quad (3)
$$

#### 4.3.2. Dedução de Relacionamento

Há três tipos de métodos de raciocínio comumente usados: tecnologia baseada em *embedding*, algoritmos de *path-ranking* e modelos gráficos probabilísticos como cadeia de Markov.

A fórmula preditiva de raciocínio relacional:

$$
Score(l, j) = \sum_{p \in Q} Path[e_l, e_j, length(p) \leq n] \cdot x_p \quad (4)
$$

onde $x_p$ é o peso do caminho alcançável $p$, de $i$ a $j$.

### 4.4. Critérios de Avaliação

Em sistemas de recuperação e extração de informação, dois indicadores principais: **Precisão** e **Recall**, e sua média harmônica chamada **F-measure**:

$$
\text{Precisão} = \frac{TP}{TP + FP} \quad (5)
$$

$$
\text{Recall} = \frac{TP}{TP + FN} \quad (6)
$$

$$
F_1 = 2 \times \frac{\text{Precisão} \times \text{Recall}}{\text{Precisão} + \text{Recall}} \quad (7)
$$

### 4.5. Resultados Experimentais

Mesclaram dados anotados de quatro fontes em um *dataset*. Construíram três modelos (NER1, NER2 e NER3):

- **NER1:** não usa *useGazettes*
- **NER2:** usa *useGazettes* com opção *cleanGazette*
- **NER3:** usa *useGazettes* com opção *sloppyGazette*

## 5. Conclusão e Trabalho Futuro

Este artigo constrói uma ontologia para cibersegurança baseada em vulnerabilidade e propõe um método para construir uma base de conhecimento de cibersegurança. O Stanford NER foi usado para treinar um extrator para extrair entidades relacionadas a cibersegurança; no entanto, **a acurácia de reconhecimento precisa de melhoria adicional**. Adicionalmente, novos atributos de entidade e novos relacionamentos entre entidades podem ser obtidos usando regras dedutivas.

**No futuro, o trabalho mais importante será enriquecer a base de conhecimento de cibersegurança e as regras de dedução, e então aplicá-los à pesquisa de detecção de intrusão e consciência situacional.**

## Resumo dos pontos-chave para o nosso paper

### O modelo quintupla — base para nossa ontologia

Jia 2018 propõe que um KG de cibersegurança seja modelado como **quintupla = ⟨Conceito, Instância, Relação, Propriedades, Regra⟩**. Esta é a estrutura formal canônica que nosso paper estende.

**Como diferenciamos:**
- Jia 2018: KG **construído estaticamente** a partir de texto (CVEs, blogs, fóruns). NER + regras dedutivas sobre dados textuais.
- Nosso paper: KG **operado em tempo de execução** sobre eventos de tráfego HTTP. Sessão como entidade semântica de primeira classe (não extraída de texto).

### Citação no nosso artigo

Já citado em `\cite{jia2018practical}`:
- **§1.2 gap 3:** "KGs construídos estaticamente a partir de texto, não como runtime"
- **§1.4 Contribuição 2:** "em contraste com grafos de cibersegurança anteriores, construídos estaticamente a partir de texto"
- **§2.1 (Grafos de Conhecimento em Cibersegurança):** quando desenvolvermos, esta é a referência metodológica primária para o quintupla

### O reconhecimento incompleto admitido pelos autores

A frase *"a acurácia de reconhecimento precisa de melhoria adicional"* é evidência de que mesmo o trabalho seminal de Jia 2018 não resolve completamente o problema de extração automática de KG de cibersegurança. Justifica nossa escolha de operar sobre eventos de tráfego estruturados (não sobre texto extraído).
